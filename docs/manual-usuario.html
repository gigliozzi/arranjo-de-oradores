1. Visão geral

O Arranjo de Oradores organiza discursos públicos e envia lembretes automáticos por WhatsApp ao orador antes da data do discurso.

O uso diário acontece pelo Django Admin. Não há uma tela separada fora do painel administrativo.

  • Cadastre oradores e congregações.
  • Mantenha a lista de temas de discursos ativa.
  • Cadastre o discurso com data, hora, orador, tema e congregação de destino.
  • Acompanhe as notificações, respostas e status das mensagens.

2. Acesso ao Admin

  1. Acesse a URL de produção do sistema.
  2. Entre em /admin/.
  3. Informe usuário e senha.
  4. Após o login, use o menu lateral para acessar os cadastros.
Atenção Use uma conta de administrador apenas com pessoas autorizadas. Quem acessa o Admin pode alterar cadastros e disparar notificações.

3. Cadastro de oradores

Use este cadastro para registrar as pessoas que receberão lembretes de discursos.

  1. No menu, acesse Oradores.
  2. Clique em Adicionar orador.
  3. Preencha nome, celular e congregação de origem.
  4. Use observações para anotações internas, se necessário.
  5. Mantenha ativo marcado para oradores disponíveis.
  6. Clique em Salvar.
Celular Informe o celular com DDD. Para envios reais pelo WhatsApp, prefira usar o padrão com código do país, por exemplo 5511999999999.

Se um orador não deve mais receber convites ou lembretes, desmarque ativo em vez de excluir o cadastro.

4. Cadastro de congregações

Use este cadastro para registrar congregações de destino e seus dados de contato/endereço.

  1. No menu, acesse Congregações.
  2. Clique em Adicionar congregação.
  3. Preencha nome, responsável e telefone, se disponíveis.
  4. No bloco Endereço, informe o CEP.
  5. Ao terminar de informar o CEP, aguarde o preenchimento automático dos campos de endereço.
  6. Revise logradouro, bairro, cidade e estado.
  7. Preencha manualmente número e complemento, quando existirem.
  8. Clique em Salvar.
Endereço na mensagem O endereço cadastrado da congregação é incluído automaticamente no lembrete enviado por WhatsApp.

5. Temas de discursos

Os temas permitem selecionar um discurso a partir de uma lista predefinida.

  1. No menu, acesse Temas de discursos.
  2. Use a busca para localizar por número ou título.
  3. Abra um tema para editar o título ou o status ativo.
  4. Desmarque ativo para temas descontinuados.
Número e título O número facilita a busca no Admin. No WhatsApp, o sistema envia apenas o título do discurso, sem o número.

Se um tema mudar de nome, atualize o título mantendo o mesmo número. Isso preserva a organização da lista.

6. Cadastro de discursos

O discurso é o agendamento principal. Ele liga orador, tema, congregação, data, hora e status.

  1. No menu, acesse Discursos.
  2. Clique em Adicionar discurso.
  3. Selecione o orador.
  4. Selecione o tema predefinido. Use o número ou parte do título para buscar.
  5. Selecione a congregação de destino.
  6. Informe data e hora.
  7. Escolha o status.
  8. Use observações para informações internas.
  9. Clique em Salvar.

Status do discurso

Status Uso recomendado
Agendado Use quando o discurso foi marcado, mas ainda não teve confirmação final.
Confirmado Use quando o orador confirmou o compromisso.
Cancelado Use quando o discurso não acontecerá mais.
Realizado Use após o discurso acontecer.
Notificações automáticas As notificações são criadas para discursos com status Agendado ou Confirmado.

7. Marcos automáticos

Ao salvar um discurso novo, o sistema cria notificações para os marcos de 30, 15, 7 e 2 dias antes da data do discurso.

Exemplo: se o discurso está marcado para 30/07, o marco de 30 dias fica previsto para 30/06.

Regra para discursos novos Marcos vencidos antes da data de cadastro não são criados. Se um discurso for cadastrado com 7 dias de antecedência, o sistema cria apenas os marcos de 7 e 2 dias.
Cron automático O cron da Railway roda automaticamente todos os dias e processa as notificações pendentes.

Notificações já enviadas permanecem registradas como histórico, mesmo que o discurso seja editado depois.

8. Consulta e reenvio de notificações

A tela Notificações mostra os lembretes criados para cada discurso.

  1. No menu, acesse Notificações.
  2. Use filtros por data, marco, status, orador, congregação ou status do discurso.
  3. Abra uma notificação para consultar detalhes do envio.
  4. Para envio manual, selecione uma ou mais notificações.
  5. No menu de ações, escolha Enviar/reprocessar notificações selecionadas.
  6. Clique em Executar.

Status de envio

Status Significado
Pendente A notificação ainda não foi enviada e pode ser processada pelo cron ou manualmente.
Enviado A mensagem foi aceita pela API de WhatsApp.
Erro Houve falha no envio. Consulte o campo de resposta da API.
Reenvio Para retestar uma notificação com erro, selecione-a e use primeiro Marcar selecionadas como pendente. Depois execute o envio.

9. Respostas e status das mensagens

O sistema registra respostas recebidas pelo WhatsApp e eventos de status das mensagens.

Respostas de notificações

  1. Acesse Respostas de notificações.
  2. Consulte telefone, nome do contato, mensagem recebida e notificação relacionada.
  3. Use esse histórico para acompanhar confirmações ou dúvidas enviadas pelo orador.

Eventos de status das mensagens

  1. Acesse Eventos de status das mensagens.
  2. Consulte status, telefone, ID da mensagem e notificação relacionada.
  3. Use esses eventos para verificar entrega, leitura ou outros retornos da API.
Webhook Respostas e status dependem da configuração correta dos webhooks da Z-API.

10. Rotina recomendada

  1. Cadastre ou atualize oradores ativos.
  2. Cadastre ou atualize congregações e endereços.
  3. Confirme se o tema do discurso está ativo.
  4. Cadastre o discurso com data e hora corretas.
  5. Verifique a aba de notificações dentro do discurso.
  6. Acompanhe diariamente notificações com status Erro.
  7. Consulte respostas recebidas para registrar confirmações.
  8. Após o evento, altere o status do discurso para Realizado.
Boa prática Use filtros e busca antes de criar novos cadastros. Isso evita duplicidade de oradores, congregações e discursos.

11. Erros comuns e o que fazer

Notificação com erro

  1. Abra a notificação.
  2. Leia o campo resposta da API.
  3. Corrija o cadastro relacionado, se necessário.
  4. Marque a notificação como pendente.
  5. Execute o envio novamente.

Mensagem não chegou

  1. Confira se o celular do orador está correto.
  2. Verifique se a notificação aparece como Enviado.
  3. Consulte Eventos de status das mensagens.
  4. Confirme se a instância do WhatsApp está conectada na Z-API.

Orador sem celular ou celular incorreto

  1. Acesse Oradores.
  2. Abra o cadastro do orador.
  3. Corrija o campo celular.
  4. Salve e reenvie a notificação, se necessário.

Discurso cadastrado com poucos dias de antecedência

Para discursos novos, o sistema não cria marcos que já venceram antes do cadastro. Se o discurso for cadastrado com 7 dias de antecedência, os marcos de 30 e 15 dias não serão criados.

Cron não processou no horário esperado

  1. Verifique se há notificações pendentes com data prevista vencida.
  2. Confira se o serviço notificacoes-cron está ativo na Railway.
  3. Consulte os logs do cron na Railway.
  4. Se necessário, faça um teste com um horário próximo.

Tema inativo não aparece na seleção

  1. Acesse Temas de discursos.
  2. Localize o tema pelo número ou título.
  3. Marque ativo.
  4. Salve e volte ao cadastro do discurso.

Webhook não registra resposta

  1. Confirme se a resposta foi enviada para o mesmo número conectado à Z-API.
  2. Verifique a URL configurada no webhook da Z-API.
  3. Confirme se o token do webhook está correto.
  4. Consulte os logs da Railway enquanto envia uma resposta de teste.
Evite apagar histórico Prefira alterar status ou corrigir cadastros. Excluir registros pode remover informações úteis para auditoria e acompanhamento.
